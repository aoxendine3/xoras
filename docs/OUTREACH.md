# XORAS Market Acquisition Strategy

This document defines the strategy for acquiring the first 5 pilot engineering teams.

## Value Proposition
"XORAS is a GitHub Action that performs static analysis on build artifacts, Docker tags, and secrets to prevent configuration-related outages and supply chain vulnerabilities."

## Communication Templates

### 1. Technical Outreach (Email/LinkedIn)
**Subject**: Feedback Request: GitHub Workflow Security Auditor

"Hi [Name], I’m developing a tool called XORAS that audits GitHub Actions for unpinned SHAs and high-risk triggers (like `pull_request_target`). It also validates build-time environment variables to prevent deployment failures. I’m looking for engineering teams to provide feedback on our audit reports—would you be open to a 10-minute technical review of our dashboard?"

### 2. Social Media Announcement (X/Twitter)
"Just released XORAS: An automated auditor for GitHub Actions. It identifies unpinned actions and workflow misconfigurations that standard CI tools miss. No external data access, runs locally. Check it out: https://aoxendine3.github.io/xoras/"

## Distribution Channels
- **Technical Communities**: Vercel, Next.js, and Docker-focused Slack/Discord groups.
- **GitHub**: Auditing trending repositories for workflow misconfigurations to provide proactive feedback.
