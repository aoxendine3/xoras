# XORAS Outreach Playbook

This document outlines the strategy for acquiring the first 5 pilot engineering teams.

## Value Proposition
"XORAS is a simple GitHub Action that prevents production outages by auditing your build artifacts, Docker tags, and security secrets before you deploy."

## Outreach Templates

### 1. LinkedIn (Governance Focus)
**Subject**: Hardening [Company Name]'s Release Integrity
"Hi [Name], I’ve been building a governance tool called XORAS that audits release candidates for release finality. It specifically catches Docker tag drift and Next.js route errors that standard CI tools miss. I'm looking for 5 engineering teams to pilot it—would you be open to a quick look at the dashboard?"

### 2. Twitter/X (Developer Focus)
"Just launched XORAS: a local-first auditor for GitHub. It stops you from pushing broken Next.js builds or unpinned Docker tags to production. Simple setup, zero-data leaks. Looking for early feedback from engineering teams. Check it out: https://aoxendine3.github.io/xoras/"

### 3. Direct Email (Technical Focus)
**Subject**: Auditing [Repo Name] for Release Finality
"Hi [Name], I noticed your team is shipping [Project/Product]. I'm building XORAS, a GitHub Action that performs 'Xero-knowledge' audits on build artifacts to ensure production grounding. It catches secret leaks and route mismatches before deployment. I’d love to get your thoughts on the audit summary we generate. Do you have 10 minutes next week?"

## Target Channels
- **Slack**: Vercel Community, CNCF, local Engineering Leadership groups.
- **Discord**: Next.js, Docker, and Security-focused communities.
- **GitHub**: Trending Next.js repositories and security tools.
